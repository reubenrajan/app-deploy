Discipline A — AgRT
Agentic Resilience Testing
Testing the resilience of AI agents themselves — validating how autonomous agents behave when their tools fail, context degrades, memory becomes corrupted, or reasoning chains collapse. The agent is the system under stress. You probe cognitive degradation, loop divergence, tool-outage recovery, adversarial prompt injection, multi-agent cascade failure, and graceful fallback behaviour across the full agent lifecycle.
AI is the resilience subject
Discipline B — ABCE
Agent-Based Chaos Engineering
Using AI agents as autonomous chaos operators — deploying agents to design, execute, and analyse chaos experiments against traditional infrastructure and services. The agent is the chaos practitioner. It generates hypotheses, selects failure modes, injects faults, observes system response, and produces post-experiment findings — dramatically accelerating and scaling traditional chaos practice.
AI is the chaos instrument

∩ Shared Foundation Skills — Required for Both Disciplines
Chaos Engineering Principles
In AgRT you apply steady-state hypothesis formation to AI cognitive failure modes; in ABCE you operationalise it through agents executing classical infrastructure chaos. Same intellectual framework, different target systems.
Chaos ToolkitGremlin
Observability & Telemetry (OTel)
OTel traces, Prometheus metrics, and structured logs are the measurement instruments for both disciplines. AgRT adds LLM-specific semantic conventions; ABCE uses classical infrastructure metrics. Neither can function without robust observability.
OTel SDKPrometheusGrafana
SLO / SLI / Error Budget Design
SLOs define when an experiment has found a real problem. In ABCE they cover latency and availability; in AgRT they must be extended to hallucination rate and task completion quality — not just classical infrastructure dimensions.
HarnessDatadog SLOs
Kubernetes & Cloud-Native Fluency
ABCE requires K8s for fault injection targets; AgRT requires K8s knowledge to inject infrastructure failures into the agent's supporting compute. Pod scheduling, resource limits, and HPA behaviour are directly relevant to both failure modes.
KubernetesLitmusChaos
FMEA & Failure Mode Analysis
ABCE applies FMEA to service topology; AgRT applies it to agent architecture components. The skill transfers directly; the vocabulary shifts from microservices and databases to planners, tool routers, and context windows.
Python Scripting & API Automation
Chaos Toolkit, LitmusChaos, and LangChain are all Python-first. Both disciplines require engineers who can wire together tool APIs, process telemetry programmatically, and build repeatable test automation without relying on GUI-only workflows.
Python asynciopytest
LLM API & Agent Architecture Literacy
Both disciplines require working knowledge of LLM APIs, tool use, and agent orchestration — either to understand what you're testing (AgRT) or to build the testing agent itself (ABCE). Understanding ReAct loops and context management is non-negotiable for either path.
LangChainClaude API
Blast Radius & Safety Discipline
In ABCE this is infrastructure scope limiting; in AgRT it means confining adversarial tests to sandboxed environments. Both disciplines are dangerous without it — autonomous agents injecting unconstrained chaos or adversarial prompts without guardrails is the nightmare scenario.
SteadybitGremlin
Distributed Systems Resilience Patterns
Circuit breakers, bulkheads, retries, timeouts, and saga patterns — the vocabulary of resilient design. ABCE tests whether these patterns work; AgRT borrows and adapts them for agentic contexts (e.g., cognitive circuit breakers that halt a looping agent).
CI/CD Pipeline Integration
Both disciplines require their experiments to run automatically on deployment events, not just quarterly GameDays. GitHub Actions, Harness, and GitLab CI integrate both chaos and resilience gates. The shift-left imperative applies equally to both.
GitHub ActionsHarness
Incident & Post-Mortem Analysis
ABCE converts post-mortems into new chaos experiments; AgRT converts agent failure incidents into new adversarial test scenarios. The feedback loop from production incidents to proactive testing is central to both practices.
PagerDuty
Statistical & Hypothesis-Driven Thinking
Both disciplines require rigorous experimental thinking: clear hypotheses, measurable outcomes, controlled variables. AgRT demands extra rigour due to non-determinism; ABCE requires rigour to distinguish meaningful resilience findings from noise introduced by the experiment itself.

When to prioritise AgRT skills
Your organisation is shipping AI agents into production where failure has real consequences — customer-facing products, internal automation with write access, or AI systems in regulated environments. You need to know agents won't hallucinate under stress, loop indefinitely, or bypass safety constraints. The practitioners who can design and run these experiments are exceptionally rare — the discipline is forming now and the frameworks are still being written.
When to prioritise ABCE skills
Your organisation has a mature or growing resilience practice and wants to dramatically scale its chaos engineering capability — running more experiments, covering more failure modes, and producing better post-experiment analysis without growing SRE headcount. ABCE is the more accessible entry point: existing chaos engineers can adopt agent tooling incrementally, the infrastructure failure vocabulary is well understood, and the tools already have the APIs agents need.
